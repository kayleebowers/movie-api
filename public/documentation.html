myFlix

Introduction

myFlix is a Node.js and Express application that provides users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints

Request URL Method
Return a list of ALL movies to the user /movies GET
Return data about a single movie by title to the user /movies/:title GET
Return data about a genre (description) by name/title /genres/:genre GET
Return data about a director (bio, birth year, death year) by name /directors/:name GET
Allow new users to register/title /register POST
Allow users to update their user info /register/:name/ PUT
Allow users to add a movie to their list of favorites /favorites/:title POST
Allow users to remove a movie from their list of favorites /favorites/:title DELETE
Allow existing users to deregister /register/:name DELETE

Endpoints in detail

Return a list of ALL movies to the user

Using http://localhost:8080/movies will return an array of movie objects in JSON format. The method type is GET and has no response body. The query requires no parameters.

Return data about a single movie by title to the user

Using http://localhost:8080/movies/:title will return a JSON object that includes information about a single movie. The method type is GET and has no response body. The query requires a movie title (Ex: The Godfather) as a parameter written without quotation marks. An example of the URL with the parameter is http://localhost:8080/movies/The%20Godfather.

Return data about a genre (description) by name/title

Using http://localhost:8080/genres/:genre will return an HTML response. The method type is GET and has no response body. The query requires a genre name (Ex: Crime) as a parameter written without quotation marks. An example of the URL with the parameter is http://localhost:8080/genres/Crime, which would return the response "So many genres to choose from."

Return data about a director (bio, birth year, death year) by name

Using http://localhost:8080/directors/:name will return a JSON response with the value of the "name" key. The method type is GET and has no response body. The query requires a director name (Ex: "Steven Spielberg") as a parameter written without quotation marks. An example of the URL with the parameter is http://localhost:8080/directors/Steven%20Spielberg, which would return the JSON response "Steven Spielberg."

Allow new users to register/title

Using http://localhost:8080/register with the POST method will return the HTML response "You are registered" or "No name found". The response body must be a JSON object, like { "name": "Cool Name"}. The query requires no parameter. Entering a valid JSON name in the request body would return the HTML response "You are registered."

Allow users to update their user info

Using localhost:8080/register/:name/ with the PUT method will return the HTML response "User {name}'s ID was updated to {ID}" or "User not found". No response body is required. The query requires a name and ID parameter. For example, localhost:8080/register/Test/3 would return the HTML response "User Test's ID was updated to 3."

Allow users to add a movie to their list of favorites

Using localhost:8080/favorites/:title with the POST method will return a JSON object with the new favorite movie information or the HTML response "No title found." The response body must be a JSON object with the following properties: {"title": "Star Wars", "director": "George Lucas", "description": "Stars goes to war", "genre": "Sci-fi", "imageUrl": "cool.jpg"}. The query requires the movie title as a parameter. The example Star Wars JSON object here would be shown in the response.

Allow users to remove a movie from their list of favorites

Using http://localhost:8080/favorites/:title with the DELETE method will return an HTML response: "The movie {title} has been deleted from your favorites." No response body is required. The query requires the movie title as a parameter. The example http://localhost:8080/favorites/12%20Angry$20Men would return an HTML response with "The movie "12 Angry Men" has been deleted from your favorites.

Allow existing users to deregister

Using http://localhost:8080/register/:name with the DELETE method will return an HTML response: "User {user.name} has been removed." No response body is required. The query requires the user name as a parameter. The example http://localhost:8080/register/Test would return an HTML response with "User Test has been removed."
