myFlix

Introduction

myFlix is a Node.js and Express application that provides users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints

Request URL Method
Return a list of ALL movies to the user /movies GET
Return data about a single movie by title to the user /movies/:title GET
Return genre description by searching genre name/title /genres/:name GET
Return data about a director (bio, birth year, death year) by name /directors/:name GET
Allow new users to register /register POST
Allow users to update their user info /register/:Username/ PUT
Allow users to add a movie to their list of favorites /register/:Username/movies/:MovieID POST
Allow users to remove a movie from their list of favorites /favorites/:title DELETE
Allow existing users to deregister /register/:name DELETE

Endpoints in detail

Return a list of ALL movies to the user

Using http://localhost:8080/movies will return an array of movie objects in JSON format. The method type is GET and has no response body. The query requires no parameters.

Return data about a single movie by title to the user

Using http://localhost:8080/movies/:title will return a JSON object that includes information about a single movie. The method type is GET and has no response body. The query requires a movie title (Ex: Spirited Away) as a parameter written without quotation marks. An example of the URL with the parameter is http://localhost:8080/movies/Spirited%20Away.

Return genre description by searching genre name/title

Using http://localhost:8080/genres/:name will return a JSON response. The method type is GET and has no response body. The query requires a genre name (Ex: Drama) as a parameter written without quotation marks. An example of the URL with the parameter is http://localhost:8080/genres/Drama, which would return the response, "In film and television, drama is a category or genre of narrative fiction (or semi-fiction) intended to be more serious than humorous in tone."

Return data about a director (bio, birth year, death year) by name

Using http://localhost:8080/directors/:name will return a JSON object with the director information. The method type is GET and has no response body. The query requires a director name (Ex: "Steven Spielberg") as a parameter written without quotation marks. An example of the URL with the parameter is http://localhost:8080/directors/Steven%20Spielberg, which would return a JSON object with Steven Spielberg's information.

Allow new users to register

Using http://localhost:8080/register with the POST method will return a JSON object showing the new user's information (as well as an automated ID). The response body must be a JSON object in the following format: { "Username": String, "Password": String, "Email": String, "Birthday": Date }. The query requires no parameter. Entering a valid JSON object in the request body will return the JSON response object. An example of a valid entry is: { "Username": "NewUser", "Password": "543", "Email": "new@email.com", "Birthday": "1960-04-29" }.

Allow users to update their user info

Using localhost:8080/register/:Username/ with the PUT method will return a JSON object displaying all of the user's updated information. An already registered username is required as the parameter, and the request must include a request body in the following format: { Username: String, (required) Password: String, (required) Email: String, (required) Birthday: Date }. For example, localhost:8080/register/existingUser could have a request body as follows: { "Username": "existingUser", "Password": "newOne", "Email": "update@email.com", "Birthday": "2023-09-09" }. A successful response returns all the existing user data, which will include the updated sections.

Allow users to add a movie to their list of favorites

Using localhost:8080/register/:Username/movies/:MovieID with the POST method will return a JSON object with all of that user's data, including their expanded favorite movies list. The query has no body but must include an existing username and a movie ID as parameters. The example http://localhost:8080/register/Coolkid/movies/649a271eef2156d71e630260 would return a JSON object displaying all of the user Coolkid's data, along with the new favorite movie.

Allow users to remove a movie from their list of favorites

Using http://localhost:8080/favorites/:title with the DELETE method will return an HTML response: "The movie {title} has been deleted from your favorites." No response body is required. The query requires the movie title as a parameter. The example http://localhost:8080/favorites/12%20Angry$20Men would return an HTML response with "The movie "12 Angry Men" has been deleted from your favorites.

Allow existing users to deregister

Using http://localhost:8080/register/:name with the DELETE method will return an HTML response: "User {user.name} has been removed." No response body is required. The query requires the user name as a parameter. The example http://localhost:8080/register/Test would return an HTML response with "User Test has been removed."
